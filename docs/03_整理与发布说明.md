# 整理与发布说明

## 1. 本交付版未纳入的内容

为了让交付目录更像“能启动、能查看、能说明”的最终版，本次没有复制以下类型的内容：

- 历史运行日志：如 `backend*.log`、`frontend*.log`、`mysql*.log`
- 临时脚本：如根目录各类 `generate_*.py`、`update_*.py`
- 旧版或中间版论文文件
- 备份 SQL：如 `online_exam_one_click_backup_*.sql`
- 辅助工程目录：如 `pythonProject`
- 临时输出目录：如 `output`
- 前端已构建产物：`OnlineExamSystemVue/dist`
- IDE 目录：如 `.idea`、`.vscode`

这些内容大多不影响项目启动，保留在原始工程里仅作为历史备份。

## 2. 如果要继续清理原始目录

当前已经生成了一份独立的最终交付版目录：

```text
D:\1毕业论文接单\OnlineExamSystem_FinalDelivery
```

因此，原始目录中的很多文件都可以在确认后删除。

建议删除候选项包括：

- 根目录历史日志
- 根目录 Python 脚本
- `pythonProject/`
- `output/`
- `__pycache__/`
- `.playwright-cli/`
- 旧版论文或重复材料
- 不再需要的备份 SQL

## 3. 公开上传到 GitHub 前必须注意

当前交付版里仍然包含以下不适合直接公开的内容：

### 3.1 配置中的敏感信息

文件：

```text
OnlineExamSystemApi/src/main/resources/application.properties
```

风险：

- 数据库账号与密码
- AI 接口地址与密钥

建议：

- 公开前将真实值替换为占位符
- 优先保留 `application.example.properties`

### 3.2 个人信息与论文材料

目录：

```text
docs/01_必须保留材料/
```

风险：

- 论文正文
- 开题报告、任务书
- 手机号、邮箱、证件号等个人信息
- 检测报告与参考文献核验表

建议：

- 如果仓库要公开，请先确认你愿意公开这些材料
- 更稳妥的做法是把这些材料留在本地交付版，不放进公开仓库

### 3.3 大体积目录

公开仓库一般不建议直接包含：

- `OnlineExamSystemVue/node_modules/`
- `apache-maven-3.8.8/`
- `OnlineExamSystemApi/target/`

建议：

- 公开版仓库保留源码与说明
- 通过 `README` 引导他人安装依赖、构建和启动

## 4. 推荐的公开版策略

如果后续要做公共 GitHub 仓库，建议拆成两个版本：

### 本地交付版

保留：

- 必须论文材料
- `node_modules`
- 自带 Maven
- 已打包 JAR

优点：

- 拿到就能跑
- 适合本地交接、答辩、老师检查

### GitHub 公开版

保留：

- 前后端源码
- SQL 脚本
- README
- 脱敏后的配置模板

移除：

- 论文材料
- 日志
- `node_modules`
- Maven 二进制
- JAR
- 真实密钥与密码
